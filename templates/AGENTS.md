# Storyline — Agent Configuration

This file primes AI coding assistants (Codex, OpenCode, Claude Code, and any others that read `AGENTS.md`) with the activation phrases and skill locations for Storyline. It's the bootstrap — the full methodology lives in the skill files.

## Activating Storyline

**In Claude Code or OpenCode:** type `/storyline` to start or resume planning. Type `/follow-up` to resolve inline `{{bracketed notes}}` in your manuscript.

**In Codex:** type `$storyline` or say "use storyline" / "start storyline" / "plan my novel". For note resolution, type `$follow-up` or say "check my notes" / "resolve my TBDs".

When activated, read the canonical skill file for your harness and execute the startup state check exactly as documented in the skill.

## Canonical skill locations

Each harness installs the skill body to a different path. Read whichever exists in this project:

- Claude Code: `.claude/skills/storyline/SKILL.md`, `.claude/skills/follow-up/SKILL.md`
- OpenCode:   `.opencode/storyline/SKILL.md`, `.opencode/follow-up/SKILL.md`
- Codex:      `plugins/storyline/skills/storyline/SKILL.md`, `plugins/storyline/skills/follow-up/SKILL.md`

The skill bodies are identical across harnesses. Only the wrapper/packaging differs.

## Project state

- `.storyline/state.json` — the canonical project state. Never edit manually.
- `.storyline/memory.jsonl` — the durable memory log. Pushed to `odd-flow` MCP on every save.
- `output/stages/` — per-stage markdown docs, auto-generated by `storyline save`.
- `manuscript/` — the writer's prose, one `.md` per chapter.

## Non-negotiables

- Character-first, always — the protagonist's wound/lie/flaw must be established before any beat-sheet work.
- Every `storyline save` writes three artefacts (state JSON, stage doc, memory log) and returns a JSON payload. The skill MUST push the memory entries to `odd-flow` via `mcp__odd-flow__memory_store` after each save.
- The skill never writes prose. The VS Code extension is where prose lives.
- When the writer wants to resolve inline `{{…}}` notes, the `/follow-up` skill scopes by (1) explicit path argument, (2) `.storyline/active-file.txt` breadcrumb, (3) `state.writing.manuscriptPath`, (4) fallback ask.

## odd-flow MCP server

The odd-flow server (configured in `.mcp.json` / `opencode.json` / `plugins/storyline/.mcp.json`) provides durable memory. On first activation the harness will prompt to approve it — approve it. Without odd-flow the skills still run but memory doesn't persist between sessions.
